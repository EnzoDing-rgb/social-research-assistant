## Page 4 · 核心理念 1

**标题：** Spec-Driven — 先定标准，再开工

**视觉：** 左边一份"研究规格说明书"（问题 + 纳入/排除标准 + 质量维度），右边箭头指向搜索结果

**内容：**
- 开源工具：**OpenSpec**（46K+ Stars，Fission-AI 团队）
- 核心理念：写代码前先写规格——在研究里就是"搜文献前先定义什么算好文献"
- 研究规格包含三样东西：
  - 研究问题可操作化（一句话 + 三个子问题）
  - 文献纳入/排除标准（出版年限、期刊等级、方法要求、样本量门槛）
  - 质量门禁（每条结论需要哪些证据支撑）

**一句话：**
你是导演，AI 是演员。剧本你写，演出 AI 来。

**来源：** 鸟窝《AI 时代的软件工程》§03 规格驱动开发 + OpenSpec 的 Delta Specs 增量规格机制

## Page 5 · 核心理念 2

**标题：** 多 Agent 交叉验证 — 三角验证的 AI 实现

**视觉：** 三个 Agent 从不同方向（学术数据库/开放网络/本地文档）独立搜索，汇聚到同一个结论，中间一个绿勾（或红叉，标注"需要人工判断"）

**内容：**
- 开源工具：**GPT Researcher**（26.6K+ Stars）+ **Academic Research Skills**（14.7K+ Stars）
- GPT Researcher 负责搜：Planner 拆解问题 → 多个执行 Agent 并行搜 → Publisher 汇总
- ARS 负责验：
  - 引用审计——每条引用抓原文验证是否真的存在
  - 多 Agent 评审——3 个 Reviewer + 1 个魔鬼代言人独立打分
  - 魔鬼代言人的**唯一任务**是挑毛病——且不轻易妥协（退让必须有证据）

**为什么有效：**
一个 Agent 可能幻觉。三个 Agent 独立验证，同时幻觉的概率极低。

**来源：** 鸟窝 §07 autoresearch（多 Agent 轮转交叉审核）+ ARS v3.0 Concession Threshold Protocol

## Page 6 · 核心理念 3

**标题：** 自主迭代 + 端到端闭环 — 不够好就自动重来

**视觉：** 一个循环箭头，标注"Ralph Loop"。箭头内：搜索 → 验证 → 不通过 → 换策略重试 → 再验证 → 通过 → 进入下一阶段。右边一条竖线串联 4 个阶段：/prd → /goal → /review-it → /ship

**上半部分 — Ralph Loop（自主迭代）：**
- 开源工具：Anthropic 官方 ralph-wiggum 插件
- 设一个质量门，不通过就自动换策略重来（最多 3 轮）
- 搜索引擎没搜够 → 换关键词、换数据源、引文追溯
- Stop Hook 防止死循环：3 轮后仍不通过 → 交给人工判断

**下半部分 — Goal Workflow（端到端闭环）：**
- 开源工具：smallnest/goal-workflow（npx 一键安装）
- 四个命令串成一条线：`/prd` → `/goal` → `/review-it` → `/ship`
- 从 PRD 到上线，每一步有明确的输入、产出和质量门

**来源：** 鸟窝 §04 Ralph Loop + §08 Goal Workflow
